      🚀 Setup & Instalação

      Como rodar o Cantinho System em servidor próprio com soberania total dos dados — opções do mais barato ao mais robusto

      🏠 Local + Raspberry Pi

      RECOMENDADO

      Soberania total + custo recorrente zero

      R$ 800 único + R$ 0/mês
      📦 Hardware: Raspberry Pi 4 8GB + cartão SD 64GB
      🌐 Backend: PocketBase (binário único, ~50MB)
      🔒 Acesso externo: Cloudflare Tunnel (grátis)
      💾 Backup: Backblaze B2 (R$ 6/mês p/ 100GB)
      • Funciona offline — internet só pra acesso remoto
      • Dados nunca saem da casa do estabelecimento
      • Conformidade LGPD máxima
      • Funciona com queda de energia se UPS

      ☁️ Cloud Free Tier

      Zero infraestrutura, mais barato

      R$ 0 /mês para uso pequeno
      🌐 Frontend: Cloudflare Pages (grátis)
      💾 Backend + DB: Supabase (500MB grátis)
      🔐 Auth: Supabase Auth (50k usuários grátis)
      📁 Arquivos: Supabase Storage (1GB grátis)
      • Sem hardware para comprar
      • Acesso de qualquer lugar automaticamente
      • Backup automático no PostgreSQL
      • Limite: 500MB DB (cabe ~5 anos de dados)

      🖥 VPS Dedicado

      Mais escalável, custo previsível

      R$ 30 /mês
      🌐 Provedor: Hostinger VPS (R$ 30) ou Contabo (R$ 50)
      📦 Stack: Node.js + PostgreSQL no Docker
      🔒 HTTPS: Let's Encrypt (grátis)
      💾 Backup: Backblaze B2 incremental
      • Controle total + recursos garantidos
      • Acesso de qualquer lugar
      • Sem limites de uso
      • Custo previsível e baixo

      🏠 Opção A — Raspberry Pi + PocketBase (passo a passo)

      1. Compras (Brasil)

      • Raspberry Pi 4 8GB — R$ 700 (FilipeFlop, Pichau)
      • Fonte 5V 3A USB-C — R$ 60
      • Cartão MicroSD SanDisk 64GB Class A2 — R$ 80
      • Case com cooler — R$ 80
      • (Opcional) UPS pequeno 600VA — R$ 250
      Total único: R$ 920–1.170

      2. Instalação do sistema operacional

      # No seu computador, baixe e grave Raspberry Pi OS 64-bit Lite no cartão SD
# Use o Raspberry Pi Imager (raspberrypi.com/software)
# Configure: SSH habilitado, usuário 'cantinho', senha forte, rede WiFi

      3. Instalar PocketBase (backend single-binary)

      # SSH no Raspberry
ssh cantinho@ip-do-raspberry

# Baixar PocketBase ARM64
mkdir ~/cantinho && cd ~/cantinho
wget https://github.com/pocketbase/pocketbase/releases/download/v0.22.0/pocketbase_0.22.0_linux_arm64.zip
unzip pocketbase_0.22.0_linux_arm64.zip

# Rodar
./pocketbase serve --http=0.0.0.0:8090

# Acessar admin em http://ip-do-raspberry:8090/_/
# Criar collections: usuarios, transacoes, produtos, etc.

      4. Servir o frontend (HTML estático)

      # Copia o index.html do Cantinho System pra o pi:
scp index.html seed-produtos.js cantinho@ip-do-raspberry:~/cantinho/pb_public/

# PocketBase serve automaticamente arquivos da pasta pb_public
# Acesso: http://ip-do-raspberry:8090/

      5. Auto-iniciar como serviço (systemd)

      sudo nano /etc/systemd/system/pocketbase.service
# Cole:
[Unit]
Description=PocketBase Cantinho
After=network.target
[Service]
ExecStart=/home/cantinho/cantinho/pocketbase serve --http=0.0.0.0:8090
WorkingDirectory=/home/cantinho/cantinho
User=cantinho
Restart=always
[Install]
WantedBy=multi-user.target

sudo systemctl enable pocketbase
sudo systemctl start pocketbase

      6. Acesso externo seguro (Cloudflare Tunnel — grátis)

      # Instala cloudflared no Pi
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-arm64 -o cloudflared
sudo mv cloudflared /usr/local/bin/ && sudo chmod +x /usr/local/bin/cloudflared

# Login + criar tunnel
cloudflared tunnel login
cloudflared tunnel create cantinho
cloudflared tunnel route dns cantinho cantinho.seudominio.com
cloudflared tunnel run cantinho

# Agora https://cantinho.seudominio.com responde com HTTPS automático

      7. Backup automático em nuvem

      # PocketBase tem backup nativo via Admin UI
# Crie um cron diário pra mandar pro Backblaze B2:
crontab -e
# Adicione: 0 3 * * * cd /home/cantinho/cantinho/pb_data && tar czf - . | b2 upload-file backup-bucket - cantinho-$(date +\%Y\%m\%d).tar.gz
      ✓ Resultado: Sistema rodando 24/7, dados 100% no seu hardware, acesso via internet seguro, backup diário automático. Custo recorrente: ~R$ 6/mês (apenas o backup B2).

      ☁️ Opção B — Supabase + Cloudflare Pages (passo a passo)

      1. Criar projeto Supabase (grátis)

      • Acesse supabase.com e clique "Start your project"
      • Login com GitHub ou Google
      • Crie organização "Cantinho do Boni"
      • Novo projeto: nome "cantinho-system", senha do Postgres forte, região São Paulo
      • Aguarde ~2min provisionamento

      2. Criar tabelas (use o SQL Editor do Supabase)

      -- Execute no SQL Editor:
CREATE TABLE usuarios (id uuid DEFAULT gen_random_uuid() PRIMARY KEY, email text UNIQUE, nome text, perfil text, modulos jsonb, status text DEFAULT 'ativo', created_at timestamptz DEFAULT now());
CREATE TABLE bank_transactions (id uuid DEFAULT gen_random_uuid() PRIMARY KEY, fitid text UNIQUE, date date, amount numeric, memo text, category text, mes_referencia text, impacta_dre boolean DEFAULT true);
CREATE TABLE caixa_transacoes (id uuid DEFAULT gen_random_uuid() PRIMARY KEY, tipo text, categoria text, valor numeric, descricao text, observacao text, operador text, created_at timestamptz DEFAULT now());
-- ... e assim para cada coleção do localStorage
-- Habilite Row Level Security (RLS) em cada tabela para segurança

      3. Configurar Auth do Supabase

      • Authentication → Providers → Email habilitado
      • Authentication → Settings → Disable "Enable signups" (só admin cria contas)
      • Crie usuários manualmente em Authentication → Users → Invite user

      4. Adaptar o frontend pra usar Supabase em vez de localStorage

      // No index.html, antes do main script, adicione:
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
<script>
  const SUPABASE_URL = 'https://SEU-PROJETO.supabase.co';
  const SUPABASE_ANON_KEY = 'sua-anon-key-aqui';
  const supabase = window.supabase.createClient(SUPABASE_URL, SUPABASE_ANON_KEY);

  // Sobrescreve o objeto Store para usar Supabase
  const Store = {
    async get(key, def){
      const { data } = await supabase.from('kv_store').select('value').eq('key', key).single();
      return data?.value ?? def;
    },
    async set(key, val){
      await supabase.from('kv_store').upsert({ key, value: val });
    }
  };
</script>

      5. Hospedar frontend no Cloudflare Pages

      • Acesse pages.cloudflare.com
      • Login + Connect to Git → suba seu repo
      • Build settings: vazio (é HTML puro), output dir: /
      • Deploy. Em ~30s tem URL cantinho.pages.dev
      • Custom domain: aponte app.cantinhodoboni.com.br nas DNS
      ✓ Resultado: Sistema acessível em app.cantinhodoboni.com.br 24/7, dados em PostgreSQL gerenciado, login real com email/senha. Custo: R$ 0/mês (gratuito até 500MB de dados, ~5 anos de uso típico).

      🖥 Opção C — VPS Dedicado (passo a passo)

      1. Contratar VPS

      • Hostinger VPS KVM 1: R$ 30/mês — 1 vCPU, 4GB RAM, 50GB SSD
      • Contabo VPS S: R$ 50/mês — 4 vCPU, 8GB RAM, 200GB SSD (melhor custo/benefício)
      • Escolha Ubuntu 22.04 LTS, datacenter Brasil ou EUA

      2. Setup inicial (após receber IP do servidor)

      ssh root@SEU.IP.AQUI

# Atualizar sistema
apt update && apt upgrade -y

# Criar usuário não-root
adduser cantinho && usermod -aG sudo cantinho

# Instalar Docker
curl -fsSL https://get.docker.com | sh
usermod -aG docker cantinho

# Firewall
ufw allow OpenSSH && ufw allow 80 && ufw allow 443 && ufw enable

      3. Stack via Docker Compose

      # Como usuário cantinho:
mkdir ~/cantinho && cd ~/cantinho
nano docker-compose.yml
# Cole:
version: '3.8'
services:
  postgres:
    image: postgres:16
    restart: always
    environment:
      POSTGRES_PASSWORD: senha_forte_aqui
      POSTGRES_DB: cantinho
    volumes:
      - ./pgdata:/var/lib/postgresql/data
  pocketbase:
    image: ghcr.io/muchobien/pocketbase:latest
    restart: always
    ports:
      - "127.0.0.1:8090:8090"
    volumes:
      - ./pb_data:/pb_data
      - ./pb_public:/pb_public
  caddy:
    image: caddy:2-alpine
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy_data:/data
volumes:
  caddy_data:

docker compose up -d
